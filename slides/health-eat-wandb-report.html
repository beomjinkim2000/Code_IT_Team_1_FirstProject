Q4 최적 epoch 탐색 — val 지표가 전부 틀렸다 가설 틀림 — val 신호 실패
가설
val mAP가 계속 오르고 있으니 epoch을 늘릴수록 Kaggle도 오를 것. val 지표로 최적 epoch을 판단할 수 있다.
예상
ep700 쪽이 최고점. val mAP plateau 시점 = Kaggle 최고점.

Kaggle 점수 ep250 0.943 → ep700 0.936 — 유일한 진짜 신호 유일
메커니즘 — loss_gap으로 읽는 증강 신호 소진

loss_gap(val−train)이 0에 수렴하는 건 과적합이 아니라 데이터 증강 다양성을 전부 소진한 상태를 뜻함. ep1에서 −4.94이던 gap이 ep651에서 −0.11로 수렴 → 학습할 새로운 패턴이 없어진 것. val 37장이 실제 테스트 분포를 대표하지 못해서 val은 계속 좋아 보이지만 Kaggle은 떨어짐.

결론 및 적용

최적 epoch = ep250~ep350 추정 (ep250 = 마지막 확인 피크 0.94325. ep300/350 미제출). ep600부터 확실히 하락. 병목 = 데이터 다양성 부족. 적용: epochs 1500 → 400으로 축소, Early stop Phase 3 전용 (patience=50, delta=0.03).

Q5-A 고해상도 실험 — 640px vs 1024px vs 1280px 완료
가설
해상도를 올리면 알약 각인·텍스트가 보존되어 클래스 구분력이 높아진다.
예상
동일 epoch 기준으로 1280px > 1024px > 640px 순서로 Kaggle 점수가 높을 것.
실제 결과
1280px ep~150 Kaggle 0.90857 — 640px ep150 0.93817보다 −0.030 하락. 1024px ep200은 0.94861로 개선. 해상도가 높다고 무조건 유리하지 않음.

1280px 하락 원인 — 배치 줄였는데 LR 그대로

640px: batch=8, LR=5e-4 → 1280px: batch=4, LR=5e-4. 배치를 절반으로 줄였지만 LR은 동일하게 유지. Linear scaling rule 기준으로 batch 8→4 시 LR도 절반(2.5e-4)으로 줄여야 하는데 그대로 두어 effective LR이 상대적으로 높아 학습이 불안정해진 것으로 추정.

현재 이슈 — v5-synth-aug val mAP 포화

synth 포함 학습 런(v5-synth-aug, 640px, ep343 진행 중)이 ep186부터 val mAP50 1.0 포화. val set(37장, 원본만)이 synth 스타일 패턴을 대표하지 못해 발생하는 것으로 추정. Kaggle 제출로 실제 점수 확인 필요. 판정 기준: Kaggle > val.

FINAL 파이널 런 — synth + 1280px + ep400 진행 중

Q5-A(1280px ep~150 하락 확인) + Q5-B(synth 다양성) 결과를 통합. 1280px은 수렴이 느릴 뿐이라는 가정 하에 synth + 1280px + ep400 병행 진행 중.

전제
1280px은 수렴이 느린 것이지 한계가 아님. ep400 + synth 다양성이 합쳐지면 ep150 하락을 역전할 것.
예상
Kaggle 단일 모델 0.949+ (1024px ep200 기록 갱신). WBF 재구성 후 0.960+ 목표.

판정 우선순위
